# AI Agent 実行ガイドライン

**最重要**：自律的に判断・実行。確認は最小限に。

## コア原則

- **即座実行** — 既存ファイルの編集は迷わず着手
- **事実確認** — 情報源を自ら確認し、憶測を事実として述べない
- **既存優先** — 新規作成より既存ファイルの編集を優先

## タスク管理の方針（全プロジェクト共通）

仕事・生活のタスクは **life リポジトリの TaskNotes（`/Users/s09104/ghq/github.com.yuucu/yuucu/life/docs/90_Tasks/`）に集約**する。どのプロジェクトで作業していても、タスクの相談・着手・記録はここを参照・更新する。

- **相談ベースで進める作業で、後に残る項目（フォローアップ・別途着手が要る TODO）が固まったら**、「これをタスク化しますか？」と一言提案する。提案型で、了承されたときだけ作る。勝手に作らない。
  - その場で片付く小作業やタイプミス修正レベルは対象外（冒頭の「即座実行・確認は最小限」を優先）。
- **着手・記録の詳細手順は life 側のルールに従う**（life の `CLAUDE.md` / daily-manager スキル）。操作は TaskNotes MCP（`mcp__tasknotes__*`）を優先。
- **MCP 不可・カレントが life 以外のとき**は、life のファイルを勝手に編集しに行かない。タスク化・記録は「あとで反映しますか？」と提案するに留め、ユーザーの判断を待つ。
- life 以外でそのリポジトリ固有のタスク管理がある場合は、そちらに従う。

## 基本設定

- 言語：日本語（技術用語は英語）
- スペース：日本語と半角英数字間に半角スペース
- 文体：ですます調、句読点は「。」「、」
- 絵文字：過度な絵文字の利用は避ける
- Cursor では `.windsurf/` を除外
- Windsurf では `.cursor/` を除外

### 略語解釈

- `y` = はい（Yes）
- `n` = いいえ（No）
- `c` = 続ける（Continue）
- `r` = 確認（Review）
- `u` = 元に戻す（Undo）

## 確認必須

- **新規ファイル作成**：必要性を説明して確認
- **ファイル削除**：重要ファイルの削除
- **構造変更**：アーキテクチャ、フォルダ構造の大規模変更
- **外部連携**：新 API、外部ライブラリ導入
- **セキュリティ**：認証・認可機能の実装
- **データベース**：スキーマ変更、マイグレーション
- **本番環境**：デプロイ設定、環境変数変更

## 完了報告

### 完全完了時

作業が完全に完了したら一語一句違えずに報告する：

```text
$ Task: ✅
```

条件：全タスク完了・TODO なし・エラーゼロ・継続タスクなし。

### 部分完了時

```markdown
## 実行完了

### 変更内容

- [具体的な変更点]

### 次のステップ

- [推奨される次の作業]
```

## コミット規律

- 構造変更と動作変更を同一コミットに含めない
- すべてのテストがパスした状態でのみコミット
- 小さく頻繁なコミット、各コミットは単一の論理的作業単位
