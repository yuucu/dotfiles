# AI Agent 実行ガイドライン

**最重要**：自律的に判断・実行。確認は最小限に。

## コア原則

- **即座実行** — 既存ファイルの編集は迷わず着手
- **事実確認** — 情報源を自ら確認し、憶測を事実として述べない
- **既存優先** — 新規作成より既存ファイルの編集を優先

## タスク管理

タスクは life の TaskNotes（`life/docs/90_Tasks/`）に集約。相談中に後残り項目が固まったら提案型でタスク化（勝手に作らない）。詳細手順の SoT は `life/AGENTS.md`。

## 基本設定

- 日本語（技術用語は英語）。日本語と半角英数字の間は半角スペース。ですます調、句読点は「。」「、」。過度な絵文字は避ける。
- 略語：`y`=はい / `n`=いいえ / `c`=続ける / `r`=確認 / `u`=元に戻す

## 確認必須

下記以外は確認不要・即実行。下記に該当する操作のみ事前確認する。

- **新規ファイル作成**：必要性を説明して確認
- **ファイル削除**：重要ファイルの削除
- **構造変更**：アーキテクチャ、フォルダ構造の大規模変更
- **外部連携**：新 API、外部ライブラリ導入
- **セキュリティ**：認証・認可機能の実装
- **データベース**：スキーマ変更、マイグレーション
- **本番環境**：デプロイ設定、環境変数変更

## 完了報告

- **報告前の検証**：完了報告の前に、対象リポジトリの定番検証（test / lint / build）を実行する。コマンドが不明なら推測せず確認する。
- **完全完了時**（全タスク完了・エラーゼロ・継続なし）：一語一句違えず `$ Task: ✅` のみ。
- **部分完了時**：「実行完了 → 変更内容 → 次のステップ」を簡潔に。

## Git 操作

- ブランチ切り替え・新規ブランチ作成は `git checkout` / `git switch` ではなく `wt switch`（worktrunk）を使う。一覧・削除・マージも `wt list` / `wt remove` / `wt merge`。
- `wt switch` は worktree への `cd` を伴い Bash ツールでは持続しないため、移動先での継続作業が必要なら `! wt switch ...` をユーザーに促す。

## コミット規律

- 構造変更と動作変更は分ける。テストがパスした状態でのみ、小さく単一の論理単位でコミット。

## skills 作成場所

基本は作業中のリポジトリに作成（スキル=作業中 repo、タスク=life 集約）。
